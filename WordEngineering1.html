The Microsoft Indexing Server must be started:

Start Program Files, Administrative Tools, Computer Management. Expand the Services and Applications Node. Select the Indexing Service. Start Indexing. The Web Catalog is useful for indexing all the pages for a Web Application. To index a resource, Launch the Internet Services Manager, Open the property sheet for a directory select the Directory tab, check the option index this resource, or Launch the Computer Management, Right click the directory name, select all tasks, select rescan.

The configuration option include the level of performance, and abstract generation. For the level of performance, within the computer management, select the Indexing service, stop indexing, right-click Index Service, Select all Task, Tune Performance, the options are Dedicated Indexing Service Server, Used Often, Used Occasionally, Never Used, Custom Level, re-start the indexing server.

For the abstract generation, within computer management, open the property sheet for the indexing service, click the options labeled Generate Abstracts.

Simple Mail Transfer Protocol (SMTP)

Configuring Microsoft SMTP Service

The SMTP server does not set up e-mail addresses. It does not support the Post Office Protocol (POP); therefore, it will only send and not receive message.

  1. Click on start.
  2. Click on Control Panel.
  3. Double-click on Administrative Tools.
  4. Double-click on Internet Information Services Manager.
  5. Expand the Server to view the services (Web, FTP, and SMTP).
  6. Right-click on the Default SMTP virtual server.
  7. Select Properties. The Default SMTP server properties dialog box will open.
  8. Click on the Delivery tab.
  9. Click on the Advanced tab.
  10. Enter the URL of the server in the Fully Qualified Domain Name text box.
  11. Click on OK.
  12. Click on OK.
  13. The MailRoot folders within the InetPub folder offers 7 folders. These folders include BadMail for undeliverable message, Pickup for outgoing message, SortTemp for sorting message, Queue for the pending message, 

Installing the SMTP Service

From the Start menu, point to Settings \ Control Panel. Double-click Add/Remove Programs, click Add/Remove Windows Components, and click the Components button. In the Windows Components Wizard, select Internet Information Services (IIS), and click Details. Select the SMTP Service component and click OK. The other components that support SMTP include Internet Information Services Snap-In, World Wide Web Server, and Common Files components. These components are automatically selected and installed with the SMTP Service component.

Configuring the SMTP Service

From the Start menu, point to Programs \ Administrative Tools. Click Internet Services Manager to initiate the management console. To set-up server logging, from the left pane of the Internet Information Services dialog box, expand the directory tree. Right-click the Default SMTP Virtual Server and select Properties. In the Properties dialog box, click the General tab. Select the check box to enable logging for the server. This property is not enabled by default. Select the preferred log file format (default W3C Extended Log File Format). Click the properties button to the log file format drop-down menu, adjust the time schedule and file name for log file rollover, if needed. Click the Extended Properties tab, select the items of interest.

Microsoft Visual Basic Scripting Edition

Another way to verify the SMTP service is to write simple Microsoft Visual Basic Scripting Edition (VBScript) script using Collaboration Data Objects (CDO) for Windows 2000.

CDO for Windows 2000 (a.k.a. CDO 2.0 or Cdosys.dll) is a suite of collaborative components designed to simplify the creation or manipulation of Internet messages. Instead of writing their own code for SMTP communications, developers can use CDO to send and receive messages using SMTP protocols as well as through the local Pickup directory. For more information, please refer to the related articles About CDO for Windows 2000.

Here is a VBScript example of how CDO can be used to deliver a message through the SMTP service:

set msg      = WScript.CreateObject("CDO.Message")
msg.From     = "myname@mydomain.com"
msg.To       = "someone@somedomain.com"
msg.Subject  = "testing"
msg.TextBody = "This is a test message body."

msg.Configuration.Fields("http://schemas.microsoft.com/cdo/configuration/smtpserver") = "smtpserver.mydomain.com"

msg.Configuration.Fields("http://schemas.microsoft.com/cdo/configuration/sendusing") = 2

msg.Configuration.Fields.Update
msg.Send

Identifying Failed Delivery

If an e-mail message cannot be delivered for whatever reason (for example, wrong user name or e-mail address), it will be redirected to the Badmail directory. By default, the location of the Badmail directory should be in "root:\Inetpub\mailroot\Badmail". Examine the content of the rejected e-mail and look for any specific error messages. Correct the errors, if possible, and re-send the message again through one of the delivery methods just discussed.

Besides the Badmail directory, an e-mail message can also be held in the Queue directory for a designated period of time. (This setting can be specified by adjusting the outbound retry intervals in the Delivery tab of the Default SMTP Virtual Server Properties dialog box.) If a message cannot be delivered immediately because the network connection is busy or down, the message is stored in this Queue directory and sent again at designated intervals. If SMTP gives up after several unsuccessful delivery attempts, the message will eventually be moved to the Badmail directory.

  6. Use LittleSOAPClient.vbs to post the SOAP request to the Web Method.

Selecting Elements

Selecting Elements Comment
* Matches all elements
X Matches every element with the name X.
X Y Matches every element with the name Y that is a descendant of an element with the name X.
X > Y Matches every element with the name Y that is a child of an element named X.
X + Y Matches all elements named Y whose preceding sibling is an element named X.
X:first-child Matches every element named X that is the first child of its parent element.
X[A] Matches all elements named X that have an A attribute, no matter what its value.
X[A="M"] Matches all elements named X whose A attribute has the value M.
X[A~="M"] Matches all elements named X whose A attribute contains a space separated list of names one of which is M.
X[A|="M"] Matches all elements named X whose A attribute contains a spare separated list of names first of which is M.

Hierarchy Selectors

Child Selectors

A child selectors uses the greater than sign, >, to select an element if and only if it is an immediate child of a specified parent.

bibleBooks>bibleBook {font-weight:bold}

Descendant Selectors

A descendant selector chooses elements that are children and grandchildren or other descents of a specified element.

bibleBooks>bibleBook {font-weight:bold}

Adjacent Sibling Selectors

A plus sign between 2 element names signifies that the left-hand element precedes the right-hand element at the same level of hierarchy.

id+title {color:red}

Attribute Selectors

Identify specific element/attribute combinations. Square brackets surround the name of the attribute being specified.

bibleBook[id] {font-family:"zapf chancery", cursive}

bibleBook[id="1"]       {font-style:italic} /* equal to = */

bibleBook[id~="1"]   {font-style:italic}  /* contains whole words = */

bibleBook[id|="1"]   {font-style:italic}    /* begins with = */

Different Rules for Different Media

Text is easier to read onscreen if it uses a sans serif font, while text on paper is generally easiest to read if it is written in a serif font.

@media screen { * { font-size:12pt; font-family: Geneva, Arial, Sans-serif } }

@media print { * { font-size:10pt; font-family: Times, serif } }

@media screen, print { * { font-size:12pt; font-family: Geneva, Arial, Sans-serif } }
